# تقرير فحص مشروع WebStart الشامل

المشروع الحالي هو **WebStart**، وهو عبارة عن منصة سحابية متكاملة لبناء المواقع والتطبيقات (SaaS Site/App Builder) اعتماداً على تقنيات الذكاء الاصطناعي التوليدي عبر واجهة دردشة تفاعلية (Chat-based builder).

---

## 1. التقنيات الأساسية (Technology Stack)

### الخلفية (Backend)
- **إطار العمل:** Laravel 11 (PHP 8.2+)
- **جسر الربط:** Inertia.js لربط Laravel مع React بسلاسة دون الحاجة لبناء واجهة برمجة تطبيقات (API) منفصلة.
- **التواصل الفوري (WebSockets):** Laravel Reverb أو Pusher لتحديثات عملية البناء الفورية.
- **قاعدة البيانات:** MySQL (مع دعم SQLite للاختبار والتثبيت السريع).
- **نظام ملفات التخزين:** يدعم التخزين المحلي والمخازن السحابية.

### الواجهة الأمامية (Frontend)
- **إطار العمل:** React 18 مع TypeScript.
- **التنسيق والتصميم:** Tailwind CSS v4.
- **مكتبة المكونات:** Radix UI للوصولية (Accessibility) العالية وتسهيل بناء المكونات التفاعلية (Modals, Dropdowns, Tooltips, Tabs, Dialogs).
- **أدوات إضافية:**
  - **Monaco Editor:** لتعديل الكود البرمجي للمشاريع مباشرة.
  - **Lexical Editor:** محرر نصوص غني ومتطور.
  - **Recharts:** لعرض الرسوم البيانية والإحصائيات في لوحات التحكم.
  - **@dnd-kit:** لدعم السحب والإفلات (Drag and Drop) ضمن واجهات البناء.
  - **lucide-react:** أيقونات عصرية للمشروع.
  - **canvas-confetti:** مؤثرات بصرية عند إتمام المهام.

## 2. بنية قاعدة البيانات (Database Schema & Models)

يحتوي المشروع على **81 ملف ترحيل (Migration)** تغطي جميع جوانب تطبيق الـ SaaS:

- **إدارة المستخدمين والصلاحيات:** جدول `users` يحتوي على حقول للبريد الإلكتروني، التحقق، الصلاحيات (`role` كـ Admin أو User)، وحالة الحساب (`status`).
- **المشاريع والملفات:** 
  - `projects`: لتخزين بيانات المواقع المبنية (UUID، والروابط الفرعية `subdomain` والنطاقات المخصصة `custom_domain` وتاريخ النشر وحالة البناء).
  - `project_files`: لإدارة الملفات والمرفقات الخاصة بكل مشروع.
- **نماذج ومزودي الذكاء الاصطناعي (AI Providers):**
  - `ai_providers`: لحفظ مزودي الـ LLM وإعدادات الربط ونماذج التسعير الخاصة بهم.
  - `user_ai_settings`: لتخزين إعدادات الذكاء الاصطناعي الفردية لكل مستخدم (مثل مفاتيح الـ API الخاصة بهم إن رغبوا في استخدامها).
  - `build_credit_usage`: لتتبع استهلاك النقاط والائتمان أثناء عمليات البناء وتوليد المحتوى.
- **الاشتراكات والخطط والمدفوعات:**
  - `plans`: لتعريف باقات الاشتراك وميزات كل باقة (الحد الأقصى للمشاريع، استهلاك الـ Firebase، التخزين، إلخ).
  - `subscriptions` & `transactions`: لإدارة الفواتير والاشتراكات النشطة للمستخدمين.
- **نظام الإحالة (Referral System):**
  - `referral_codes`, `referrals`, `referral_credit_transactions`: لتتبع الإحالات ومنح رصيد للمستخدمين الذين جلبوا مستخدمين جدد.
- **إدارة المحتوى وتعديل الصفحة الرئيسية:**
  - `landing_sections`, `landing_items`, `landing_contents`: للتحكم في الصفحة الرئيسية وتغيير محتواها ديناميكياً من لوحة التحكم.

## 3. أهم الخدمات البرمجية (Core Services)

يحتوي مجلد `app/Services` على منطق العمل الأساسي للمشروع:

1. **`InternalAiService.php`**:
   - الخدمة المسؤولة عن التواصل مع نماذج الذكاء الاصطناعي (مثل OpenAI, Anthropic, Grok, DeepSeek).
   - تقوم بتوليد مقترحات لبناء المواقع، العناوين الرئيسية، النصوص التوضيحية للصفحة الرئيسية بشكل ديناميكي ومؤتمت بناءً على لغة المستخدم مع ميزة التخزين المؤقت (Caching).
2. **`BuilderService.php`**:
   - تتواصل مع سيرفر بناء المواقع الخارجي (Go Builder) عبر بروتوكول HTTP لبدء جلسات بناء كود للمواقع وتتبع حالتها وإرسال الأوامر.
3. **`PluginManager.php`**:
   - المسؤولة عن تحميل وتفعيل الملحقات (Plugins) بشكل ديناميكي وخاصة بوابات الدفع (Payment Gateways).
   - تدعم التثبيت المباشر للملفات المضغوطة وإضافتها إلى مسار تحميل PSR-4 ديناميكياً دون الحاجة لإعادة تشغيل خادم الويب أو تحديث ملفات Composer يدوياً.
4. **`FirebaseService` & `FirebaseAdminService`**:
   - تتيح ربط المشاريع المنشأة بقواعد بيانات Firebase / Firestore مستقلة لحفظ بيانات مستخدمي تلك المشاريع وتخزين ملفاتهم.
5. **`DomainSettingService` & `DomainVerificationService`**:
   - للتحقق من النطاقات المخصصة (CNAME / A Records) التي يربطها العميل بموقعه والتأكد من توجيهها بشكل صحيح.

## 4. الإضافات المدمجة (Installed Plugins)

يحتوي مجلد `app/Plugins/PaymentGateways` على البوابات الجاهزة للدفع والاشتراكات:
- **PayPal**
- **Stripe**
- **YooKassa** (للدفع الروسي)
- **Razorpay** (للهند)
- **Paystack** (لأفريقيا)
- **Crypto.com** (للدفع بالعملات الرقمية)
- **BankTransfer** (الدفع التحويلي البنكي)

## 5. نظام التثبيت والتشغيل والتحديث

- **`autosetup.sh`**: سكربت Bash ضخم ومفصل ومكتبوب باحترافية لتثبيت المشروع بالكامل على خوادم Ubuntu VPS. يقوم بتثبيت Nginx, PHP-FPM, MySQL 8.0, Node.js 20، وإعداد سيرفر البناء (Builder Server) على منفذ مخصص، وتوليد المفاتيح والشهادات الأمنية تلقائياً.
- **`InstallController.php`**: واجهة رسومية على الويب ترشد المسؤول للتحقق من متطلبات الخادم وصلاحيات الملفات، ثم ربط قاعدة البيانات وإنشاء الحساب الإداري الأول للموقع.
- **`UpgradeController.php`**: للتعامل مع تحديثات قاعدة البيانات وترقية المشروع إلى إصدارات أحدث.
